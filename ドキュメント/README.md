## ソースコード構成（現行）

本プロジェクトの本番実装は **静的 HTML / バニラ JS** + **Supabase（コンテンツ取得のみ）** の構成です。
旧 Next.js 実装（`src/app/exercise/...`）は廃止され、現在は `fittness_type-main/` のみが本番ソースです。

```
fittness_type-main/
├── index.html                      # メインエントリ（SPA ハッシュルーティング）
├── privacy.html                    # プライバシーポリシー
├── app.js                          # アプリ本体（IIFE。診断ロジック・UI・シェア）
├── content-loader.js               # Supabase からのコンテンツ取得とフォールバック
├── clarity-init.js                 # Microsoft Clarity 解析の初期化
├── supabase-config.js              # Supabase URL / anon key
├── tokens.css / components.css / styles.css   # スタイル
├── types-data.js                   # 16 アニマルの静的フォールバックデータ
├── secret-compat-detail-data.js    # シークレット相性データ
├── chars-data.js                   # キャラクター画像メタ
├── data.js                         # 設問/設定の静的フォールバック
├── OGP.png / favicon.png / logo.png
├── robots.txt / sitemap.xml
└── character/                      # SVG アセット
```

### 起動・運用

- **ローカル起動**: `npx serve fittness_type-main` で表示された URL を開く（API 不要）
- **本番デプロイ**: リポジトリ直下から `npx vercel deploy --prod --yes`（プロジェクト: `fit-animal.com`）
- **ハッシュルーティング**: `#/`（ウェルカム）・`#/types`・`#/types/<CODE>`・`#/quiz`・`#/result`・`#/result/<CODE>`・`#/secret-compat/<CODE>`
- **データソース**: コンテンツは Supabase の `exercise_animals` / `exercise_animal_profiles` / `exercise_questions` 等から取得。失敗時は静的 JS にフォールバック
