# 運動タイプ診断 v2 — 新診断仕様書

本ドキュメントは、MBTIおよび16Personalitiesを参考に設計した「4軸32問・16タイプ」新診断のバックエンド実装仕様です。

---

## 1. 旧診断との違い

| 項目 | 旧診断（v1） | 新診断（v2） |
| :--- | :--- | :--- |
| 軸数 | 8軸（連続スコア） | 4軸（二値判定） |
| 問数 | 40問 | 32問 |
| タイプ判定 | 軸スコア＋欲求スコアの複合計算 | 4軸の二値組み合わせで一意決定 |
| 回答形式 | 4択リッカート | 7択（16Personalitiesスタイル） |
| 設計思想 | 欲求スコア方式 | 性格気質方式（MBTI型） |
| タイプ数 | 16タイプ | 16タイプ |

---

## 2. 4軸の定義

| 軸番号 | 軸名 | マイナス極 | プラス極 | 参照フレームワーク |
| :---: | :--- | :--- | :--- | :--- |
| 軸1 | 社会性 | S（Solo：ひとり完結） | G（Group：誰かと一緒） | MBTI E/I |
| 軸2 | 構造性 | F（Flow：直感・柔軟） | P（Plan：計画・記録） | MBTI J/P |
| 軸3 | 耐性 | A（Assertive：立ち直り早い） | T（Turbulent：深く感じる・理想高い） | 16Personalities A/T |
| 軸4 | 強度性 | C（Calm：ゆるめ・安定） | D（Drive：追い込み・挑戦） | Sensation Seeking（運動固有） |

**タイプコードの構成:** 軸1軸2軸3軸4 の順（例: SFAC, GPAD）

## 4. スコアリングと判定ロジック

### 4.1 回答値の変換

```
7択の選択肢 → スコア変換

とても賛成      → +3
賛成           → +2
どちらかといえば賛成 → +1
どちらともいえない  →  0
どちらかといえば反対 → -1
反対           → -2
とても反対      → -3
```

### 4.2 軸スコアの計算

```
各軸のプラス方向（G/P/T/D）への回答をそのまま加算
マイナス方向（S/F/A/C）への回答はスコアを反転して加算

例）軸1 Q01（Solo方向）の回答が「とても賛成(+3)」の場合
  → Soloへ+3 = Groupスコアへ-3として処理

各軸の合計スコア範囲: -24 〜 +24
```

### 4.3 二値判定

```
軸1: スコア ≥ 0 → G（Group） / スコア < 0 → S（Solo）
軸2: スコア ≥ 0 → P（Plan）  / スコア < 0 → F（Flow）
軸3: スコア ≥ 0 → T（Turbulent） / スコア < 0 → A（Assertive）
軸4: スコア ≥ 0 → D（Drive） / スコア < 0 → C（Calm）

スコアがちょうど0の場合:
  その軸の8問のうち、プラス方向への回答数が多い側を採用
  それでも同数の場合: プラス側（G/P/T/D）を採用
```

### 4.4 タイプコード決定

```
軸1結果 + 軸2結果 + 軸3結果 + 軸4結果 を結合してタイプコード生成

例: Solo + Plan + Assertive + Drive → SPAD
```

### 4.5 スコアの正規化（表示用）

```
各軸のスコアを -100 〜 +100 に正規化して結果画面のバーチャートに使用

normalized = Math.round(rawScore / 24 * 100)
```

## 6. DBスキーマ（3テーブル）

### 6.1 `exercise_v2_questions`（設問マスター）

```sql
CREATE TABLE exercise_v2_questions (
  id            SERIAL PRIMARY KEY,
  question_text TEXT         NOT NULL,
  axis          SMALLINT     NOT NULL CHECK (axis IN (1, 2, 3, 4)),
  -- 1=社会性, 2=構造性, 3=耐性, 4=強度性
  plus_direction VARCHAR(1)  NOT NULL,
  -- 軸1: 'G', 軸2: 'P', 軸3: 'T', 軸4: 'D'
  minus_direction VARCHAR(1) NOT NULL,
  -- 軸1: 'S', 軸2: 'F', 軸3: 'A', 軸4: 'C'
  order_index   SMALLINT     NOT NULL,
  created_at    TIMESTAMPTZ  DEFAULT NOW()
);
```

### 6.2 `exercise_v2_types`（タイプマスター）

```sql
CREATE TABLE exercise_v2_types (
  code                    VARCHAR(4)   PRIMARY KEY,
  -- 例: 'SFAC'
  type_name               TEXT         NOT NULL,
  tagline                 TEXT         NOT NULL,
  -- A2: キャッチコピー
  concept                 TEXT         NOT NULL,
  -- A3: タイプ概要（300〜500字）
  catchphrase             TEXT         NOT NULL,
  -- A4: 口癖（1個）
  exercise_definition     TEXT         NOT NULL,
  -- B1: あなたにとっての運動の定義
  recommended_activities  JSONB        NOT NULL,
  -- B2: おすすめ運動Top5 ["活動1", "活動2", ...]
  bad_fit                 JSONB        NOT NULL,
  -- B3: Bad Fit ["やってはいけない運動1", ...]
  habit_design            TEXT         NOT NULL,
  -- C1: 継続の仕掛け（Tiny Habits構造）
  pitfalls                JSONB        NOT NULL,
  -- C2: 落とし穴 ["落とし穴1", "落とし穴2", ...]
  motivation_triggers     JSONB        NOT NULL,
  -- C3: モチベーションのスイッチ ["スイッチ1", ...]
  insight_text            TEXT         NOT NULL,
  -- D1: 見透かされ感テキスト（200字）
  reframe_points          JSONB        NOT NULL,
  -- D2: 弱みを強みに変える視点 ["視点1", ...]
  compatible_types        JSONB        NOT NULL,
  -- D3: 相性タイプ [{"code": "GFAC", "name": "盛り上げ隊長", "reason": "..."}]
  share_text              TEXT         NOT NULL,
  -- E1: Xシェアテキスト（140字以内）
  created_at              TIMESTAMPTZ  DEFAULT NOW()
);
```

### 6.3 `exercise_v2_diagnoses`（診断結果）

```sql
CREATE TABLE exercise_v2_diagnoses (
  id           UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id      UUID         REFERENCES auth.users(id) ON DELETE SET NULL,
  -- ログイン前は NULL、ログイン後に紐付け
  answers      JSONB        NOT NULL,
  -- [{"q": 1, "v": 2}, {"q": 2, "v": -1}, ...] (Q番号 + 変換済スコア)
  axis_scores  JSONB        NOT NULL,
  -- {"social": 12, "structure": -8, "resilience": 4, "intensity": -15}
  type_code    VARCHAR(4)   NOT NULL REFERENCES exercise_v2_types(code),
  created_at   TIMESTAMPTZ  DEFAULT NOW()
);

CREATE INDEX idx_exercise_v2_diagnoses_user_id ON exercise_v2_diagnoses(user_id);
CREATE INDEX idx_exercise_v2_diagnoses_type_code ON exercise_v2_diagnoses(type_code);
```

## 7. API エンドポイント設計

### POST `/api/exercise-v2/diagnose`

**リクエスト:**
```ts
{
  answers: number[]  // Q1〜Q32の回答（+3〜-3, 32要素）
}
```

**処理フロー:**
```
1. answers を受け取り、各軸のスコアを計算
2. 4軸を二値判定してタイプコードを決定
3. exercise_v2_types からタイプデータを取得
4. exercise_v2_diagnoses に結果を保存（user_id は nullable）
5. レスポンスを返却
```

**レスポンス:**
```ts
{
  diagnosisId: string,       // UUID
  typeCode: string,          // "SFAC"
  typeName: string,          // "気まぐれ散策者"
  axisScores: {
    social: number,          // -24〜+24
    structure: number,
    resilience: number,
    intensity: number
  },
  typeData: {                // exercise_v2_types の全フィールド
    tagline: string,
    concept: string,
    catchphrase: string,
    exerciseDefinition: string,
    recommendedActivities: string[],
    badFit: string[],
    habitDesign: string,
    pitfalls: string[],
    motivationTriggers: string[],
    insightText: string,
    reframePoints: string[],
    compatibleTypes: { code: string, name: string, reason: string }[],
    shareText: string
  }
}
```

## 8. 結果画面の構成定義（MVP）

### 8.1 通常ビュー（全項目表示）

```
1. タイプ名 + コード + キャッチコピー（A1, A2）
2. 4軸スコアバー（A5）
   - S/G バー（左: Solo, 右: Group）
   - F/P バー
   - A/T バー
   - C/D バー
3. タイプ概要（A3）
4. 口癖（A4）
5. あなたにとっての運動の定義（B1）
6. おすすめ運動 Top5（B2）
7. やってはいけない運動スタイル（B3）  ← シェアビューでは非表示
8. 継続の仕掛け（C1）
9. 落とし穴（C2）                      ← シェアビューでは非表示
10. モチベーションのスイッチ（C3）
11. 見透かされ感テキスト（D1）
12. 弱みを強みに変える視点（D2）
13. 相性タイプ（D3）
14. シェアボタン（E1のテキストを使用）
```

### 8.2 シェアビュー制限（isSharedView=true）

- B3（Bad Fit）: 非表示
- C2（落とし穴）: 非表示
- シェアボタン: 非表示 →「自分も診断してみる」ボタンのみ表示
